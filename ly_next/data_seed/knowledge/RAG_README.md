# 知识库（RAG）里建议放什么

本目录由配置项 `agent.rag.documents_path` 指向（默认 `data/ly_next/knowledge`）。**只放你愿意被模型检索到的文本**（`.md` / `.txt`）。

## 适合放入的内容

- **产品 / 项目说明**：功能列表、版本说明、架构一页纸。
- **接口与约定**：内部 API 路径、鉴权方式、错误码、环境变量说明。
- **运维与排障**：常见报错、检查步骤、联系人或工单入口（勿放密码）。
- **术语表**：团队缩写、业务名词中英文对照。
- **写作风格**：回答语气、禁用词、输出格式（也可部分写在仓库根目录的 `MEMORY.md` 里作为「长期记忆」）。

## 不适合放入的内容

- 密钥、Token、数据库口令、客户隐私原文。
- 超大二进制或整库代码拷贝（应写摘要 + 链接到仓库路径）。

## 与 `MEMORY.md` 的分工

- **`MEMORY.md`（根目录）**：短、稳定、全局规则与用户偏好；每次对话会注入（有长度上限 `agent.memory.max_chars`）。
- **`data/ly_next/knowledge/`**：可较长的参考文档；按**当前用户问题**做相似度检索，只拉相关片段。

模型还可通过内置工具 **`remember_fact`** 把用户要求记住的简短事实追加到 `MEMORY.md`（勿存敏感信息）。
